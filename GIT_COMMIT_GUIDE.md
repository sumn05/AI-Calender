# 📝 Git 커밋 가이드

## 커밋 메시지 컨벤션

### 형식
```
<type>: <subject>

<body> (선택사항)

<footer> (선택사항)
```

### Type 종류
- `feat`: 새로운 기능 추가
- `fix`: 버그 수정
- `docs`: 문서 수정
- `style`: 코드 포맷팅, 세미콜론 누락 등 (코드 변경 없음)
- `refactor`: 코드 리팩토링
- `test`: 테스트 코드 추가/수정
- `chore`: 빌드 업무 수정, 패키지 매니저 설정 등

### 예시

#### 1. 초기 프로젝트 설정
```
feat: 프로젝트 초기 설정 및 기본 구조 생성

- HTML 기본 구조 생성 (index.html, detail.html)
- 기본 스타일시트 추가 (styles.css)
- LocalStorage 기반 데이터 저장 구조 구현
```

#### 2. 캘린더 기능 구현
```
feat: 캘린더 뷰 기능 구현

- 월별 캘린더 렌더링
- 날짜 선택 기능
- 이전/다음 월 네비게이션
- 선택된 날짜의 이벤트 표시
```

#### 3. 이벤트 관리 기능
```
feat: 이벤트 CRUD 기능 구현

- 이벤트 추가 기능
- 이벤트 수정 기능
- 이벤트 삭제 기능
- 이벤트 목록 렌더링
- 제목, 메모, 날짜, 시간 필드 지원
```

#### 4. 링크 관리 기능
```
feat: 링크 저장소 기능 구현

- 링크 추가/삭제 기능
- 아이콘 선택 기능
- 링크 그리드 뷰
- 이벤트에 링크 연결 기능
```

#### 5. 상세보기 페이지
```
feat: 상세보기 페이지 구현

- 별도 페이지로 상세보기 기능
- 드래그 가능한 카드 레이아웃
- 메모/녹음 통합 카드
- 사진 업로드 및 관리
- 링크 관리 카드
```

#### 6. 드래그 앤 드롭
```
feat: 카드 드래그 앤 리사이즈 기능

- 카드 드래그 기능
- 4방향 리사이즈 핸들
- 확장/축소 기능
- 레이아웃 자동 저장
```

#### 7. 음성 녹음 기능
```
feat: 음성 녹음 및 전사 기능

- Web Speech API 통합
- MediaRecorder API를 활용한 오디오 녹음
- 실시간 음성 인식
- 전사 텍스트 표시
```

#### 8. AI 요약 기능
```
feat: Google Gemini API를 활용한 AI 요약

- Gemini API 통합
- 여러 모델 fallback 지원
- API 키 관리 기능
- 요약 결과 표시
```

#### 9. 운세 기능
```
feat: 일일 운세 확인 기능

- 생년월일 및 별자리 입력
- 운세 점수 계산
- 운세 상세 설명 생성
- 운세 결과 모달 표시
```

#### 10. UI/UX 개선
```
style: 모달 및 버튼 UI 개선

- 그라데이션 버튼 스타일
- 모달 애니메이션 효과
- 호버 효과 개선
- 반응형 디자인 개선
```

#### 11. 로그인 기능
```
feat: 전화번호 기반 로그인 기능

- 로그인 모달 구현
- 전화번호별 데이터 분리
- 자동 로그인 기능
- LocalStorage 기반 인증
```

#### 12. 버그 수정
```
fix: 상세보기 페이지에서 링크 동기화 문제 해결

- 메인 페이지에서 추가한 링크가 상세 페이지에 즉시 반영되지 않던 문제 수정
- setInterval을 통한 주기적 데이터 동기화 추가
- window focus 이벤트를 통한 데이터 재로드
```

## 커밋 예시 시나리오

### 시나리오 1: 새 기능 추가
```bash
git add index.html script.js
git commit -m "feat: 운세 확인 기능 추가

- 생년월일 및 별자리 입력 모달
- 운세 점수 계산 알고리즘
- 운세 결과 표시 모달"
```

### 시나리오 2: 버그 수정
```bash
git add detail.js
git commit -m "fix: 카드 레이아웃 저장 문제 해결

- 상세 페이지 재진입 시 카드 크기 초기화 문제 수정
- todos 배열 참조 유지로 변경사항 즉시 반영"
```

### 시나리오 3: UI 개선
```bash
git add styles.css
git commit -m "style: 모달 및 버튼 UI 개선

- 그라데이션 배경 적용
- 호버 애니메이션 추가
- 그림자 효과 개선"
```

### 시나리오 4: 문서화
```bash
git add README.md PROJECT_STRUCTURE.md
git commit -m "docs: 프로젝트 문서화

- README.md 작성
- 프로젝트 구조 문서 추가
- 기능 설명 및 사용법 추가"
```

## 커밋 단위 구분 원칙

1. **기능별로 구분**: 하나의 커밋은 하나의 완성된 기능만 포함
2. **논리적 단위**: 관련된 변경사항을 함께 커밋
3. **테스트 가능**: 커밋 후 즉시 테스트 가능한 상태여야 함
4. **의미 있는 단위**: 너무 작지도 크지도 않은 적절한 크기

## 브랜치 전략 (선택사항)

```
  ├── develop

  │   ├── feature/todo-management
  │   ├── feature/detail-page
  │   └── feature/recording
  └── hotfix/bug-fix
```

## 좋은 커밋 메시지 예시

✅ **좋은 예시**
```
feat: 이벤트에 링크 연결 기능 추가

- 이벤트 모달에 링크 추가 필드 구현
- 링크 아이콘 선택 기능
- 이벤트와 링크 연결 로직
- 링크 표시 UI 개선
```

❌ **나쁜 예시**
```
수정
변경사항
fix
update
```

## 참고 사항

- 커밋 메시지는 한글로 작성 가능
- 첫 줄은 50자 이내로 간결하게
- 본문은 72자 단위로 줄바꿈
- 명령형 문장 사용 (과거형 X, "추가함" X → "추가" O)
- 이슈 번호가 있으면 footer에 추가: `Closes #123`
